## 一、基本概念辨析

### 1.集群与节点

+ ES中的**集群**是一种抽象关系，是由一组节点组成的逻辑概念，它代表了**多个节点** 共同协作、协调工作的一组系统。

+ **节点** 是 **物理或虚拟机器** 上运行的 Elasticsearch 实例。每个节点都有一个 **唯一的名称**，并且可以承担不同的角色。

  **每个节点** 可能运行在 **不同的物理机器或虚拟机** 上；也可以 **多个节点运行在同一台物理机器** 上。

❓**问题：**在本地9200端口运行的这个，到底是集群？节点？还是Elasticsearch 搜索引擎？

答：是指向 **Elasticsearch 节点** 上的 **RESTful API** 接口。默认情况下，Elasticsearch 会监听 **9200 端口** 来接受 HTTP 请求。这些请求可以是查询、索引、更新等操作，通过 Elasticsearch 的 **RESTful API** 与之交互。**9200 端口** 实际上是一个 **节点** 上的端口，并且它提供的是 **REST API 接口**，允许你与该节点进行交互。
